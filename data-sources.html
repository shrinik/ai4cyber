Data Sources

This section summarizes the open source intelligence data sources that are relevant to the hospital industry

Phishtank

Phishtank is a free community site where anyone can submit, verify, track and share phishing data. It is operated by Cisco Talos Intelligence Group (Talos).

Description

  • Phishing was identified as one of the most common threats faced by the hospital industry.
  • Successful phishing attemps can lead to data breaches and ransomware which can put hospital data, patient data and patient medical records at risk.
  • PhishTank allows users to submit suspected phishes and track the status of their submissions. Multiple users must submit a phish in order for it to be added to the database.
  • PhishTank continuously updated with new threats
  • PhishTank monitors all industries, not just healthcare thus providing a comprehensive view of potential phishing attacks.

Collection Strategy

  • Phishtank data is updated hourly.
  • The data is downloaded as a CSV file from the PhishTank website.
  • This approach allows for data analysts without programming knowledge to view and manipulate the data.

Data Collection Summary

  • Number of records: 59,256
  • Date run: Week of October 6,2022
  • First Submission Data in 2008.

Shodan

Shodan is a search engine that lets users search for various types of servers (webcams, routers, servers, etc.) connected to the internet using a variety of filters. Shodan is used widely in cybersecurity research community

Description

  • The healthcare sector has a disproportionately high number of data breaches and hence data breaches was identified as a major threat trend for the hospital industry
  • Nowadays, a typical hospital utilizes dozens of IoT devices, including wearables, guided imagery, monitoring sensors, implantable, ingestible, etc.
  • Large-scale use of IoT in healthcare involves the processing and transfer of patients personal information and raises issues around privacy and security since many medical IoT devices lack robust security.
  • Shodan will help to search and identify all medical IoT devices for the organization for further assessment.
  • Shodan will be used to search and collect data related to IoT medical devices, Hospital management systems, or medical devices manufacturer that are connected to the internet directly.

Collection Strategy

  • Shodan's API will be used to search for the relevant device data using search queries.
  • DICOM is a protocol used in medical imaging machines. The search query will search for the keyword "DICOM Server Response" for port 104
  • The data is downloaded using the Shodan API and converted to a suitable format for analysis file

Data Collection Summary

  • Number of records: 1394
  • Date run: Week of October 8,2022

Common Vulnerabilities and Exposures (CVE)

The Common Vulnerabilities and Exposures (CVE) system provides a reference-method for publicly known information-security vulnerabilities and exposures. The MITRE Corporation, maintains the system, with funding from the US National Cyber Security Division

Description

  • The hospital industry's critical assets include IT infrastructure and Information Systems.
  • These assets are vulnerable to malware such as ransomware which was identified as one of the top threats faced by the hospital industry
  • CVEs that are not addressed can be exploited by adversaries to introduce malware like ransomware or for data breaches.
  • CVEs , is the abbreviation for a list of openly reported computer security issues. A security issue with a CVE ID number is what is meant when someone mentions a CVE.Almost invariably, vendors' and researchers' security warnings list at least one CVE ID.
  • CVEs aid IT specialists in the hospitals in organizing their efforts to prioritize and fix these flaws to strengthen the security of computer systems across the hospitals network.
  • Therefore it is important for the hospital industry to keep track of CVEs to understand and protect the IT infrastructure and Information systems from ransomware and data breaches.

Collection Strategy

  • We have downloaded the csv file listing all CVEs from the Mitre website

Data Collection Summary

  • Total CVE Records: 185770
  • Date run: Week of October 3,2022
