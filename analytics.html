Analytical Approaches

This section discusses the different analytical approaches used to develop AI4Cyber for the hospital industry

Supervised Machine Learning Classification

We used supervised machine learning technique to create a model that can classify a website as a phishing website or a benign website.

Overview

  • Objective: Classify websites as phishing or benign
  • Data Sources: The list of phishing and benign URLs was obtained from Phishtank & AlexaRank respectively in csv format.
  • Method: Supervised predictive analysis using classification.
  • Result: Multiple classification models were evaluated with Random Forest model having the best performance.

Justification of chosen analytical approach

  • Why this analytical technique?: The data set provided by Phishtank contains known Phishing web URL links and targets. Thus, it can be used for identifying potential phishing sites that may compromise the hospital network and data. The goal is to tease out attributes of a Phishing site so we are better able to detect phishing sites overall.
  • Industry Value:Phishing is the most effective way for a bad-actor to gain access to a hospital's network. With deeper understanding of the attributes that phishing sites have, an organization can proactively prevent attacks by deploying mitigating controls such as firewalls or SPAM filters; protecting the enterprise.
  • AI4Cyber Value: Phishtank data repositories are updated continually. Linking to this data allows for a realistic view of the phishing threat landscape and allows AI to derive ways in which defenses can pivot to better detect and block attacks.

Passive Vulnerability Assessment

Identify Common Platform Enumerations (CPEs) for devices used in hospital industry using Shodan and then search for vulnerabilities impacting CPEs using National Vulnerability Database (NVD).

Approach

  • Find internet connected devices/machines used in hospital industry using Shodan
  • Identify the common CPEs
  • Correlate the CVEs for the CPEs from above step

Justification of chosen analytical approach

  • Why this analytical technique?:Identify common vulnerabilities affecting medical devices/machines used in hospitals.
  • Industry Value:Data breaches are a common threat to hospital industry. Knowing possible vulnerabilities, hospital IT departments can address them before they can be exploited to gain unauthorized access to the hospital network resulting in data breaches.
  • AI4Cyber Value:Shodan continually crawls the internet and NVD is updated with the latest vulnerabilities. Using them in combination will result in timely and actionable intelligence on vulnerabilities in medical machines/devices exposed to the internet.

Unsupervised Machine Learning Classification

We used Weka platform for the AI analytics part and tried different models to explore and classify the data

Approach

  • Data prepared outside Weka
  • Explore, classify, and try different models in Weka

Justification of chosen analytical approach

  • Why this analytical technique?:it allows classifying medical devices/machines into different groups which can aid in further analysis w.r.t vulnerabilities and other aspects.
  • Industry Value:It will help find vulnerabilities in medical devices based on the group they fall into.
  • AI4Cyber Value:Proactively address the threat of possible vulnerabilities in medical equipment before they are found by hackers and researchers.

Unsupervised Clustering of Ransomware

Identify common features of ransomware using clustering so that they can be used to identify newer ransomware that exhibit similar features.

Approach

  • Use Kaggle dataset with 60K records across 18 attributes
  • Prepare the data
  • Feature selection
  • Classification
  • Clustering

Justification of chosen analytical approach

  • Why this analytical technique?:Allows identifying and grouping of ransomware features which may not have been found using manual analysis
  • Industry Value:Ransomware attacks are one of common threats in hospital and healthcare industry targeting patient medical records and information systems.
  • AI4Cyber Value:Allows identification of possible ransomware code by comparing its characteristics against those exhibited by ransomware
